# Недостающие данные для RetailCRM

При переносе логики RetailCRM из старого Vue проекта в новый проект были обнаружены недостающие данные, которые были заменены на моковые значения.

## Моковые данные (подставлены автоматически):

### 1. Данные доставки
- **deliveryCity**: `'Москва'` (в старом проекте бралось из формы выбора города)
- **deliveryType**: `'СДЭК до пункта выдачи'` (в старом проекте выбирался тип доставки)
- **deliveryPoint**: `null` (в старом проекте выбирался пункт выдачи СДЭК)
- **deliveryAddressInfo**: `null` (в старом проекте заполнялся адрес доставки: улица, дом, квартира)
- **mailAddress**: `''` (в старом проекте заполнялся адрес для Почты России)
- **curierAddress**: `''` (в старом проекте заполнялся адрес для курьерской доставки по СПб)
- **deliveryComment**: `''` (в старом проекте можно было добавить комментарий к доставке)

### 2. Способ оплаты
- **paymentType**: `'Банковской картой'` (в старом проекте можно было выбрать способ оплаты)

## Что нужно добавить в новый проект:

1. **Форма выбора города доставки** - для получения `deliveryCity`
2. **Форма выбора типа доставки** - для получения `deliveryType`:
   - СДЭК до пункта выдачи
   - Постамат OmniCDEK
   - СДЭК курьером до двери
   - Почта России 1 класс
   - Доставка курьером по Санкт-Петербургу
3. **Форма выбора пункта выдачи** - для получения `deliveryPoint` (если выбран СДЭК)
4. **Форма адреса доставки** - для получения `deliveryAddressInfo` (если выбрана курьерская доставка)
5. **Поле комментария к доставке** - для получения `deliveryComment`
6. **Выбор способа оплаты** - для получения `paymentType`:
   - Банковской картой
   - Долями от Тинькофф
   - СБП (Система быстрых платежей)

## Где находятся моковые данные:

Файл: `frontend/src/widgets/configurator/ui/order-popup/OrderPopup.tsx`

Строки 64-75: объект `mockDeliveryData` содержит все моковые значения для доставки
Строка 124: моковое значение для `paymentType`

## Переменные окружения:

В файле `.env` в папке `backend/` нужно добавить:
```
RETAIL_CRM_API_TOKEN=your_retailcrm_token
RETAIL_CRM_API_URL=your_retailcrm_url
```

Замените `your_retailcrm_token` и `your_retailcrm_url` на реальные значения из старого проекта.
